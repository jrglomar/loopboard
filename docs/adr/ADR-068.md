# ADR-068: The ticket generator can create a Dev task only (v1.57)

## Status: Accepted (2026-07-16) — implemented in v1.57

## Context

Since v1.17 (ADR-028) the ticket generator is PO-first: it creates a PO story by default, with an
"Also create a linked Dev task" checkbox to opt into the PO+Dev pair. There was no way to create just a
**standalone Dev task** — teams often need one (tech debt, chores, spikes, ops work) with no PO story
behind it, and had to either fabricate a PO story or leave the app. The user asked for a "Dev ticket only"
option on the creation feature.

## Decision

- **A 3-way create mode replaces the boolean toggle.** `type CreateMode = "po" | "pair" | "dev"` with a
  native `<select>` (`CreateModeSelect`, per ADR-009's native-select convention): **PO story only**
  (default, unchanged), **PO + linked Dev task** (the old checkbox-on behaviour), **Dev task only** (new).
  Both the AI-chat and the local-template forms show the same control.
- **Dev-only creates a standalone Dev task** via a new `createDevTicketOnly` wrapper in
  `hooks/useJira.ts` — `create_dev_ticket` with **no `linkedPoTicketKey`** (the tool's link has always
  been optional) and the form's **story points now going to the Dev task** (they went to the PO story in
  the other modes). Sprint targeting uses the Dev sprint select (two-sprint flow) or the single legacy
  select.
- **The UI follows the mode.** The PO sprint select and the PO draft pane hide in Dev-only mode
  (`DraftPreview` gains `showPo`, defaulting true); the success panel's PO link renders only when a PO
  ticket was created (`SuccessState.po` is now optional, mirroring what v1.17 did for `dev`).
- **No backend/contract change** — `create_dev_ticket` already supported exactly this call shape
  (optional link, optional points since v1.30, optional sprint); this is frontend wiring only.

## Consequences

- Standalone Dev tasks are first-class: drafts (AI or template) still generate both PO and Dev text, but
  only what the mode selects is shown and created — switching modes after drafting just changes which
  panes/tickets apply.
- The delegation guard is unaffected: `create_dev_ticket` is already in `JIRA_WRITE_TOOLS`, so read-only
  shared-credential users get the same 403 in every mode.
- Tests: the pair tests' `enableDevTask()` helper now sets the select to "pair" (one-place change); two
  new tests prove Dev-only calls `createDevTicketOnly` (and neither PO path), hides the PO pane/link, and
  carries story points + the selected sprint. TicketGen.test 27 → 29.
- Gate (shared with ADR-069, v1.57): typecheck + build clean, react-app 793 → 796 tests, total 1,396,
  smoke 55/55 (no server edits). Pill → 1.57.0.

## Alternatives considered

- **A second checkbox ("skip the PO story").** Two booleans with an invalid combination (neither ticket);
  a 3-value mode makes the impossible state unrepresentable.
- **Creating a Dev task linked to an existing PO story.** Already exists — that's the Linking page and
  Planning's LinkDevTicketCard (ADR-021/022); this ADR is specifically the *unlinked* case.
- **Radio group instead of a select.** Three options in two dense toolbars; the select matches the
  adjacent sprint selects and costs less width.
