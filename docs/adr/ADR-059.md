# ADR-059: UI/UX review pass ‚Äî accessibility & contrast fixes (v1.48)

## Status: Accepted (2026-07-11) ‚Äî 8 of 9 findings applied; 1 deferred by decision

## Context

An expert UI/UX review (the planned "tester" phase) audited every surface for accessibility, contrast,
state coverage, copy and consistency. Contrast was computed from the theme tokens in `globals.css` against
WCAG 2.1 AA, not eyeballed. Nine findings resulted; the full report is the published artifact. This ADR
records the fixes and, importantly, the token rationale so the colour changes aren't a mystery later.

## Decision

Applied (v1.48):

- **A11Y-01 ‚Äî the assistant popup is now a real modal.** `AssistantWidget` was `role="dialog"`
  `aria-modal="true"` but never trapped focus or closed on Escape. Kept the bespoke overlay (a Radix
  migration risked the ChatPanel's persisted-conversation behaviour) and added: focus moves into the panel
  on open and restores to the FAB on close, Tab is trapped, Escape closes. Covered by a new test.
- **A11Y-02 ‚Äî `--success` darkened `142 71% 45%` ‚Üí `142 66% 30%`.** As *text* on white the old green was
  2.30:1 (fails AA); it's used as `text-success` in 13 files. The token now measures **5.06:1**. The one
  solid fill using it (the DoD progress bar) simply reads a deeper, still-correct green. This is why
  "success" green is darker across the app now.
- **A11Y-03 ‚Äî amber "shared credentials" pills** darkened `text-amber-600` ‚Üí `text-amber-800` (2.59:1 ‚Üí
  6.31:1). These carry security meaning (who can write to Jira), so legibility matters.
- **A11Y-04 ‚Äî `--muted-foreground` `47%` ‚Üí `43%` L** so secondary text clears AA on the cool app
  background (4.35:1 ‚Üí 5.03:1), not only on white cards.
- **COPY-01 / A11Y-05 ‚Äî the app-wide login** now says "Sign in to **Loopboard**" (was "Task Helper", stale
  since v1.45) and leads with a single `<h1>`. Live-verified in the running app.
- **CODE-01 ‚Äî dead `AuthGate`** removed; `LoginForm` moved to its own file
  (`components/task-helper/LoginForm.tsx`); `AuthGate.test.tsx` deleted (AppGate.test covers the gate).
- **A11Y-06 ‚Äî reduced motion.** A global `@media (prefers-reduced-motion: reduce)` rule damps animations
  and transitions.

Deferred (by decision, not omission):

- **THEME-01 ‚Äî the dark theme has no switch.** A complete `.dark` palette exists (`darkMode: ["class"]`)
  but nothing toggles the class, so no user reaches it. Adding a toggle is a product decision (placement,
  persistence, does auto-`prefers-color-scheme` cross the app's single-tenant board expectations) and an
  untested theme flip is riskier than the polish is worth right now. Left explicitly pending.

## Consequences

- Every automated contrast pair the review flagged now clears WCAG AA, verified numerically before shipping.
- The assistant is keyboard- and screen-reader-operable, closing the one real a11y hole.
- Gate: 1,344 tests (react-app 755 ‚Äî net ‚àí3 from deleting the 4 dead AuthGate tests, +1 Escape test),
  typecheck clean, build clean, smoke 53/53. Pill stays 1.48.0 (no API surface change).
- **Next:** a focused pass 2 should drive the gated data pages (Huddle, Planning, Reports, Linking) live
  for responsive + empty/loading/error coverage this sweep couldn't reach without credentials.
