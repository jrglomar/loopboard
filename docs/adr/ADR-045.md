# ADR-045: Full sprint-report CSV export (form + pulled data)

## Status: Accepted (2026-07-01)

## Context

Reports already exports a per-assignee CSV (`buildReportCsv`) and a Markdown document. The user wants a
**full sprint-review export**: a CSV that combines data the app already has (duration, goals, points,
fly-ins) with **qualitative retro answers the user types** (team name, scrum master, what worked well,
kudos, …) — for pasting into a sprint review / retro record.

## Decision

Frontend-only.

- **Pure `buildSprintReviewCsv(report, form, flyIns)`** in `reportMarkdown.ts` (reuses the existing
  `csvCell`/`csvRow` helpers) emits a **Field/Value CSV** (two columns, one row per field) — chosen over a
  wide single-row CSV because the qualitative fields are freeform paragraphs that read better in a Value
  cell. Rows: Sprint, Duration (start–end + working days via `sprintWorkingDays`), Sprint goals,
  Team name\*, Scrum master\*, Commitment points\* (prefilled from `committedPoints`), Completed points,
  Incomplete points (`committed − completed`), Reason for delays\*, Fly-ins, What worked well\*,
  What didn't\*, Planned improvements\*, Kudos\* (\* = from the form).
- **`SprintReviewExport.tsx`** — a controlled `Dialog` form (matching `ConfirmActionDialog`) opened by a
  new **"Full report (CSV)"** button in the Reports `ExportBar`. It computes the fly-in list itself
  (`matchFlyIn` over `report.completed` + `report.notCompleted`), builds the CSV on submit, and downloads
  it (Blob). No persistence — the form resets each open (commitment reseeded from the report).

## Consequences

- One click produces a complete, shareable sprint-review CSV mixing live data + the team's own retro
  notes, with zero backend change (no tool, no store, smoke unchanged).
- The retro answers are not saved between exports (out of scope). A future enhancement could persist them
  per sprint (e.g., alongside the post-scrum store) so they prefill next time.

## Alternatives considered

- **Wide single-row CSV** (a column per field). Rejected — long paragraphs across many columns are
  unwieldy; Field/Value reads like a document.
- **Markdown/PDF instead of CSV.** The user asked for CSV; a Markdown variant could be added later beside
  the existing `buildReportMarkdown`.
- **Persist the form per sprint.** Deferred to bound scope (export-only for now).
