# ADR-053: Collapsible Huddle sidebar cards

## Status: Accepted (2026-07-07) — implemented; verified live (toggle + persistence)

## Context

The Huddle (Dashboard) right sidebar stacks seven widgets — Needs attention, Meeting goal,
Meeting notes, Impediments, On leave, Code review, and the Daily Huddle digest. On an active
sprint the column gets long, and not every team cares about every card every day. The user asked
for the cards to be collapsible. The app already had a per-card collapse precedent (the Fly-in
card, ADR-032/v1.32).

## Decision

- **Per-card collapse** (chosen by the user over a single whole-sidebar toggle), matching the
  Fly-in precedent: a chevron in each card's own header toggles that card's body.
- **`useCollapse(key, defaultCollapsed=false)`** (`hooks/useCollapse.ts`) — collapse state
  persisted to `localStorage` under `loopboard.collapse.<key>`, defaulting to **expanded** so
  nothing hides unexpectedly and existing tests (which render fresh) are unaffected.
- **`CollapseToggle`** (`components/CollapseToggle.tsx`) — a reusable disclosure button
  (chevron + the card's own icon/title as children), kept **inside each card's `<h3>`** so the
  heading role and existing text queries survive. `aria-expanded` conveys state; no `aria-label`
  so the accessible name stays the card title.
- Applied to all six compact cards + the Daily Huddle digest (digest gates its main success
  view; its transient loading/error/empty states keep their plain header). Cards with a header
  action (Meeting goal / Meeting notes "Edit") move that action beside the toggle — a button
  can't nest in the toggle button — and hide it while collapsed.
- Presentation-only: **no tool, HTTP, or contract IO change.** react-app pill → **1.43.0**.

## Consequences

- Each card gained ~4 lines (hook + toggle wrapper + `{!collapsed && …}` gate); the collapse
  logic itself lives once in the hook + toggle. `localStorage` failures (private mode) fall back
  to session-only collapse.
- The Daily Huddle's collapse only applies once data has loaded (the loaded branch is the one
  with the toggle); collapsing while loading/empty is a non-goal.
- Verified live against the real sprint: every card renders a chevron, clicking collapses the
  body (`aria-expanded=false`, content removed), and the choice **survives a page reload**
  per-card and independently.

## Alternatives considered

- **One toggle for the whole sidebar** — simpler, but all-or-nothing; the user explicitly wanted
  per-card control.
- **A wrapper `CollapsibleCard` in the Dashboard** (no per-card edits) — rejected: each card owns
  its `<Card>`+title, so an outer wrapper would either duplicate the title or overlap the cards'
  existing header actions. Integrating the toggle into each header is the clean result.
