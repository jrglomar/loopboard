# ADR-074: Team trends workbook carries the per-developer aggregate section (v1.62)

## Status: Accepted (2026-07-18) — implemented in v1.62

## Context

Since v1.59 the multi-sprint markdown/CSV exports have included the cross-sprint per-assignee
aggregate table (§4.29 `byAssignee`: Assignee · Sprints active · Done pts · Total pts · Avg
done/sprint). v1.61 introduced the styled .xlsx exports — but the "Team trends" workbook carried
only the per-sprint rows + TOTAL/AVERAGE, so the aggregate view the user relied on existed in the
plain exports and not in the styled one they now reach for first. The user asked for it back,
"but it should remain styled xls".

## Decision

`teamTrendsAoa` gains a **BY DEVELOPER** section below the AVERAGE row: section title, the five
columns above, one row per `report.byAssignee` entry. Deliberate properties:

- **Backend numbers verbatim** — `avgDonePoints` is the FULL-window velocity convention (a quiet
  sprint drags the average down), and the section is NOT leave-adjusted: that view belongs to the
  Developer KPIs workbook (ADR-072/073). One export = one convention; mixing both into a single
  table is how numbers stop being trusted.
- **Raw numerics** (Excel can sum/average them) — the file's existing convention.
- **"Unassigned" filtered** (v1.61 item 176: a ticket state, not a developer).
- Styling copies the file's own idioms (dev-block header treatment, header/data row passes); the
  layout stays a pure, index-reporting function so tests need no xlsx parsing.

## Consequences

- The styled team workbook now answers "who delivered what across the window" without opening the
  CSV; the three export formats carry the same aggregate.
- Tests: trendsXlsx 17 → 22 (section placement, order + decimal avg, Unassigned exclusion, empty
  window, index consistency); react-app 943 → **948**, total **1,582**; smoke 55/55. Pill → 1.62.0.
- Not live-eyeballed (login gate); the layout indices are pinned structurally.

## Alternatives considered

- **A second sheet** instead of a same-sheet section. The sprint-review workbook precedent is
  single-sheet sections; one printable page beats tab-hunting for a five-column table.
- **Leave-adjusted columns here too.** Rejected — duplicates the Developer KPIs workbook and
  makes "avg done" mean two different things in one file.
