# ADR-042: Linking keeps the PO title and inherits the PO's story points

## Status: Accepted (2026-06-30)

## Context

The Linking page bulk-creates Dev tasks from existing PO stories (ADR-022/025). Today the AI
(`plan-dev-tickets`) drafts **both** a new `devSummary` (title) and a `devDescription` for each Dev task.
The team wants the linked Dev task to **carry the same title as the PO story** — only the **description**
should be drafted/enhanced — and to **inherit the PO story's story points** so the Dev task is estimated
the same as its parent.

## Decision

- **Keep the PO title.** Every drafted plan item's `devSummary` is forced back to the source PO story's
  summary via a small `withPoTitle` mapper, applied to the AI result, the deterministic template
  fallback, and the AI-error fallback. The AI's own title is ignored; the AI now effectively only drives
  the **description**. Per-draft **Regenerate** (ADR-023) updates only `devDescription` (the title stays
  the PO's). The title field is still editable (relabelled "Title (kept from PO — edit if needed)").
- **Carry the PO's points — as an editable field (revised same day per feedback).** `create_dev_ticket`
  gains an optional `storyPoints` (written to `JIRA_STORY_POINTS_FIELD` exactly like `create_po_ticket` —
  `createIssue` already supported the field). The Linking plan card shows an **editable Points input
  drafted from the PO story's `storyPoints`** (plan item `storyPoints?`, set by `draftFromPo`); the user
  can override it before creating. **Create all** / **Retry failed** send the (possibly edited) plan-item
  value to `create_dev_ticket`.

## Consequences

- Linked Dev tasks are titled and estimated consistently with their PO stories, with the description the
  only AI-drafted field — matching how the team wants traceable, like-for-like Dev tasks.
- One small backend addition (`storyPoints` on `create_dev_ticket`); the AI endpoints are unchanged (the
  frontend simply overrides the title and ignores the AI's `devSummary`).
- If a PO story has no points (`storyPoints: null`), nothing is sent and the Dev task is created without
  points — same as before.

## Alternatives considered

- **Change `plan-dev-tickets` to return only a description.** Rejected — a bigger backend/AI change for
  no extra benefit; overriding the title client-side is simpler and keeps the endpoint reusable.
- **Re-fetch each PO's points at create time** (`get_ticket`). Rejected — the PO tickets are already
  loaded on the page with `storyPoints`; an extra fetch per row is wasteful.
