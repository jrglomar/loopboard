# ADR-069: The styled sprint-review Excel export excludes the offset balance (v1.57)

## Status: Accepted (2026-07-16) — implemented in v1.57

## Context

The styled .xlsx sprint-review export (v1.38, ADR-048) rendered the shared per-member table
(`buildMemberReviewTable` + `MEMBER_COLUMNS`) with all 9 columns — including **"Offset bal."**, each
developer's cumulative offset standing from the ledger. The user asked to remove the offset balance from
the exported styled Excel: the workbook is shared outward as sprint-review material, and offset balances
are internal team bookkeeping (ADR-038/061/065), not review content.

## Decision

- **Scope: the xlsx only.** `sprintReviewXlsx.ts` renders a local `XLSX_MEMBER_COLUMNS =
  MEMBER_COLUMNS.filter(c => c !== "Offset bal.")` (8 columns): the header, each member row, the TOTAL
  row, and the column-width array all drop the balance cell. The **printable HTML/PDF export keeps the
  column** (the request named the Excel; both renderers share the same model, so the HTML is untouched
  by construction).
- **Membership is unchanged.** The `ledger` parameter still feeds `buildMemberReviewTable`, so *who*
  appears in the table (the union of roster, points, leaves, and offset standing) is identical — only the
  displayed balance column is gone from the workbook.

## Consequences

- The exported workbook is 8 columns (Member → Leave days); no offset figures leave the app via Excel.
- The shared model (`sprintReview.ts`) is untouched — no risk to the HTML export, the on-screen table, or
  their tests.
- If the HTML/PDF should also drop the column later, that's a one-line change in `MEMBER_COLUMNS` +
  `memberRowCells`/`memberTotalCells` — deliberately not done here (out of the request's scope).
- Tests: the xlsx layout tests assert the 8-column header without "Offset bal.", 8-cell member + TOTAL
  rows, and the leave-days cell that now ends each row. sprintReviewXlsx.test 6 → 7.
- Gate (shared with ADR-068, v1.57): typecheck + build clean, react-app 793 → 796 tests, total 1,396,
  smoke 55/55. Pill → 1.57.0.

## Alternatives considered

- **Remove the column from the shared `MEMBER_COLUMNS`** (HTML + xlsx both). Broader than asked; the
  printable review may still legitimately show the balance internally.
- **Drop the `ledger` parameter from the xlsx call.** Would also remove ledger-only members from the
  table (membership is a union that includes offset standing) — a silent behaviour change beyond the
  request; keeping the parameter keeps rows identical.
