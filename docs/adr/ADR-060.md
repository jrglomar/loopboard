# ADR-060: In-app user guide (v1.49)

## Status: Accepted (2026-07-11) — implemented in v1.49

## Context

Loopboard grew a lot of surface — Huddle, Planning, Offset Tracker, Linking, Reports, Task Helper (with a
notes/to-do journal), Connections, an admin console, and a floating AI assistant. `docs/USER-GUIDE.md`
documents all of it, but a new teammate landing in the app has no way to find it. The user asked for a
how-to-use guide, reachable from the UI.

## Decision

- **A `Guide` page** (`pages/Guide.tsx`) presents the guide inside the app: a short intro, a table of
  contents, and one card per surface (Getting started, Huddle, Planning, Offset Tracker, Linking, Reports,
  Task Helper, Connections, Admin, the AI assistant, and Tips). The copy mirrors `docs/USER-GUIDE.md` but
  is written for reading in-app, from the user's side ("open Connections and add your token", not "set
  `TOKEN_ENC_KEY`"). The TOC is a sticky sidebar on wide screens and a plain anchored list on mobile.
- **Reached from a header button, not a tab.** The v1.48 UI review flagged the tab bar as already tight
  (7 tabs + Admin). A guide is a low-frequency destination, so it lives as a `Guide` button in the header's
  right cluster — the conventional home for help — rather than taking prime tab real estate. The button
  carries `aria-current="page"` when active; the board controls hide on it like the other non-board pages.
- Content is a `SECTIONS` array (`{ id, title, icon, body }`) so adding or reordering a section is a
  one-place edit, and the TOC is generated from it (a section can't be listed without existing).

## Consequences

- Every user (not just admins) can learn the app without leaving it; admins get the extra Admin section,
  which is labelled "visible to admins only" rather than conditionally hidden — simpler and still honest.
- The guide is **static content that duplicates `docs/USER-GUIDE.md`.** They can drift. Kept deliberately:
  the in-app copy is intentionally shorter and UI-facing, and a build-time import of the markdown would pull
  a parser/renderer in for little gain. When a feature changes, both should be updated — noted here so it's
  a known maintenance point, not a surprise.
- Gate: 1,348 tests (react-app 755 → 759: +3 Guide, +1 App button test), typecheck clean, build clean.
  react-app pill → 1.49.0.

## Alternatives considered

- **A ninth nav tab** — most discoverable, but worsens the exact crowding the UI review called out.
- **A modal / drawer** — fine for a tooltip, cramped for a multi-section guide with a TOC.
- **Render `USER-GUIDE.md` at build time** — single source of truth, but adds a markdown pipeline and the
  in-app voice should differ from the repo doc's setup-heavy voice anyway.
