# ADR-032: App-wide compact visual refresh

## Status: Accepted (2026-06-25)

## Context

The user asked to "reconsider our UI design, making compact design but clean look" — an app-wide
direction, not a single screen. The Huddle daily cards were already tightened (ADR-031); the rest of
the app (Planning, Linking, Reports, and every shadcn card) still used the roomy defaults (`p-6`
card padding, `space-y-6` page rhythm, `text-2xl` page headings).

The constraint: this is a **presentation-only** pass. No DOM structure, element semantics, text, or
behavior may change — the existing content/behavior test suite must stay green.

## Decision

A density pass via shared tokens + className tweaks only:

- **Card padding** (`components/ui/card.tsx`): default `CardHeader` / `CardContent` / `CardFooter`
  padding `p-6 → p-4`. This is the single highest-leverage change — every card that doesn't override
  padding gets tighter at once. The Huddle cards (ADR-031) set explicit `px-3` padding, which wins
  via `tailwind-merge`, so they're unaffected.
- **Page rhythm**: top-level page containers unified to `space-y-4` (Planning `space-y-6`, Linking
  `space-y-5`, Reports `space-y-6` + report `<article>`), so all four tabs share one vertical rhythm.
- **Headings**: Reports page `h1` and the per-sprint report `h2` `text-2xl → text-xl` for a tighter,
  consistent header scale.

No component is restructured; only Tailwind utility classes change.

## Consequences

- One consistent, denser, cleaner look across Huddle / Planning / Linking / Reports without touching
  markup or copy. The full test suite (content + behavior) remains the regression guard and stays
  green.
- `CardTitle`'s `text-2xl` default is left untouched — it has no usages in the app (cards use custom
  `<h3>` headers), so changing it would be inert.
- Future density tweaks have a clear home: the card primitive + the per-page container class.

## Alternatives considered

- **Restructure layouts / merge sections.** Out of scope and risky — would churn DOM and break tests;
  the ask was "compact + clean," not a re-layout.
- **A global root `font-size` reduction.** Too blunt — scales everything including already-correct
  type; padding + rhythm are the parts that actually read as "roomy."
- **Per-page bespoke redesigns.** Inconsistent and high-effort; the shared card primitive gives an
  app-wide result from one edit.
