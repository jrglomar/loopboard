# ADR-076: Total rebrand — Loopboard → InvokeBoard (v1.64)

## Status: Accepted (2026-07-18) — implemented in v1.64

## Context

The user renamed the product to **InvokeBoard** and asked for a total rebrand: every detail in the
app, the working folder, and the GitHub repo. This repo has done this once before (Agile AI
Companion → Loopboard, 2026-06-16), and that rename's playbook and traps drove this one.

## Decision

**Casing**: user-facing = "InvokeBoard" (the banner keeps its two-tone split as
`Invoke<span>Board</span>`); every identifier = lowercase `invokeboard` (npm scope
`@invokeboard/*`, Docker images `invokeboard/*`, volume `invokeboard-data`, store files
`.invokeboard-*.json`, per-user dir `.invokeboard-user-stores/`, localStorage prefix
`invokeboard.`, session cookie `ib_session`).

**What was renamed**: UI strings and titles, all three package names + the root package (lockfile
regenerated via `npm install`, never edited by hand), Docker artifacts, config.ts store-file
DEFAULTS, .gitignore patterns, scripts/smoke labels, and every LIVING doc (README, CLAUDE.md,
CONTRACTS.md in full — it describes the current system — SETUP, DEPLOYMENT, ARCHITECTURE,
USER-GUIDE, ASSISTANT, plus the auxiliary briefs). The developer's real host-local store files
were migrated by `mv` only — data preserved, files never opened (the users store holds encrypted
credentials). GitHub repo → `jrglomar/invokeboard` and folder → `C:\Projects\invokeboard`
(old GitHub URLs redirect; the open PR stack survives a repo rename).

**What was deliberately NOT renamed**:
- **ADR-001..075** — dated decision records keep the name that was true when they were written
  (the same principle the Loopboard rename applied to historical narrative).
- **Jira's `/rest/agile/1.0/` paths** — Atlassian's API, not a brand string (the standing rule
  from the first rename).
- **`.env`** — never opened (tokens); explicit `*_FILE` overrides keep working since absolute
  paths win over the renamed defaults.

**Accepted one-time resets**: localStorage keys (`loopboard.*` → `invokeboard.*`) reset persisted
UI prefs like card collapse; the session cookie rename (`lb_session` → `ib_session`) signs
everyone out once.

## Consequences

- Zero contract-surface change beyond identity: tool names, routes, ports, and error codes are
  untouched; tests hold at **1,589** (assertions updated in place), smoke 55/55, pill → 1.64.0.
- **Grep has a blind spot for split-tag brand strings.** AppGate.tsx carried its own
  `Loop<span>board</span>` that no text search could see; it was caught only by loading the real
  login screen in a browser. Rule for the next rename: after the sweep, eyeball every surface
  that renders the brand, and grep for the fragment halves (`>board`, `Loop<`) too.
- **Repo-hygiene finding (pre-existing, surfaced by the rename): the real meeting-notes store
  was committed once** — `packages/mcp-jira/.loopboard-meeting-notes.json` in commit `d884591`
  on pushed `main`, despite being gitignored (it was force-added or predates the ignore rule).
  The rename stops tracking it from v1.64 forward, but the content remains in the repo's pushed
  history. No credentials are in that store (meeting notes only), so this is a privacy judgment,
  not a key rotation: if the repo is private and the notes are innocuous, leaving history alone
  is reasonable; scrubbing requires `git filter-repo`/BFG plus a force-push that rewrites every
  open PR's base — a deliberate human decision, deferred to the user.
- The folder rename must be done by the harness's own PowerShell host in one command
  (`Set-Location C:\Projects; Rename-Item loopboard invokeboard`) — an outside shell holding the
  cwd blocks it ("Device or resource busy", the documented trap from the first rename).

## Alternatives considered

- **Rename ADR history too.** Falsifies dated records for zero operational gain; a reader of
  ADR-020 should see the world as it was.
- **Sed the lockfile.** The regenerated lockfile is the only trustworthy one; hand-edited
  lockfiles are how phantom-dependency bugs are born.
- **Migrate localStorage/session state.** A migration shim for cosmetic prefs and a 7-day cookie
  is more code than the inconvenience it removes.
